Work & Contributions

A selection of backend modules I've built as part of a maritime crew management system — covering data architecture, cross-module integration, and procurement workflows.

System Module

Seafarer Preparation

Displays seafarer and crew information sourced from our database, with records aggregated across multiple tables to ensure accurate, complete details. It includes built-in editing functionality for updating seafarer information as needed, and serves as the central entry point into several other modules across the system.

System Module

Working Gears — Ordering

Processes the required personal protective equipment (PPE) orders for seafarers and trainees, pulling seafarer details directly from the Seafarer Preparation module. It displays the full list of working gear needed for training, along with the supplier, price, available sizes, and currency used for each purchase.

Automation

CTM Automation

Automates the importation of records from an external Oracle database into our local database on a monthly schedule. The workflow fetches CTM payment data per crew and per vessel through HTTP requests, then triggers an email notification once all records have been successfully updated. The synchronized data is subsequently consumed by the accounting module for financial processing.

Automation

Speech-to-Text Generator

Transcribes audio recordings and generates concise summaries for documentation and future reference — particularly useful for recalling meeting discussions. Built using two different approaches: one using AssemblyAI, which leverages multiple API endpoints for uploading, transcribing, and retrieving transcript sentences before summarizing via a DeepSeek chat model; and a second, more streamlined approach using Gemini AI, which achieves the same result with significantly fewer steps.
